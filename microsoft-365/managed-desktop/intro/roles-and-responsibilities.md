# <a name="microsoft-managed-desktop-roles-and-responsibilities"></a>Microsoft Managed Desktop ról i obowiązków

<!--This topic is the target for a "Learn more" link in the Admin Portal (aka.ms/admin-access); do not delete.-->
<!-- from Roles and responsibilities -->

Co firma Microsoft robi dla Ciebie, gdy Twoja organizacja Microsoft Managed Desktop usługi w twojej organizacji? Jaki jest zakres obowiązków Twojej organizacji?

## <a name="microsofts-roles-and-responsibilities"></a>Role i obowiązki firmy Microsoft

Firma Microsoft zapewnia następujące kluczowe role i obowiązki:

| Rola lub odpowiedzialność | Opis |
| ----- | ----- |
| Zarządzanie zasadami MDM | Firma Microsoft zastosuje zasady MDM zgodnie z najlepszymi rozwiązaniami i rozważy żądania zmiany zasad. Ponadto dokonamy zmian w Twojej dzierżawie w sposób określony w [zasadach urządzenia](../service-description/device-policies.md). |
| Pomoc techniczna dla użytkowników | W razie potrzeby udostępnimy mechanizm uzyskiwania podwyższonym poziom dostępu do urządzeń i eskalowania problemów poprzez wniosek o pomoc techniczną. Aby uzyskać więcej informacji, zobacz [Obsługa użytkowników](../service-description/user-support.md).
| Microsoft Managed Desktop pomocy technicznej usługi | Firma Microsoft będzie zapewniać pomoc techniczną Twojeowi działowi IT za pośrednictwem Microsoft Managed Desktop Operations Team. Ten zespół będzie obsługiwać rozwiązywanie problemów technicznych, żądania zmiany i zarządzanie zdarzeniami w środowisku Microsoft Managed Desktop klienta. Aby uzyskać więcej informacji, zobacz [Pomoc techniczna dla administratorów Microsoft Managed Desktop](../working-with-managed-desktop/admin-support.md). |
| Monitorowanie zabezpieczeń | Firma Microsoft będzie monitorować Twoje urządzenia Microsoft Managed Desktop za pomocą programu Microsoft Defender for Endpoint. Jeśli centrum Microsoft Managed Desktop security operations Center (SOC, Security Operations Center) wykryje zagrożenie, powiadomimy Cię, wyizoluje urządzenie i zdalnie poprawimy problem. Aby uzyskać więcej informacji, zobacz [Zabezpieczenia](../service-description/security.md). |
| Monitorowanie i zarządzanie aktualizacjami | Aktywnie monitorujemy Twoje urządzenia Microsoft Managed Desktop, aby zapewnić, że na urządzeniach firmy Microsoft Windows i Microsoft Microsoft Office są zainstalowane najnowsze aktualizacje jakości i Microsoft Office. Aby uzyskać więcej informacji, zobacz [Jak obsługiwać aktualizacje](../service-description/updates.md). |
| Grupowanie użytkowników i urządzeń | Microsoft Managed Desktop tworzyć wymagane grupy użytkowników i urządzeń oraz zarządzać nimi w ramach operacji IT. Te grupy nie mogą zmieniać członkostwa ani konfiguracji. Zmiana tych grup może doprowadzić do nieoczekiwanej konfiguracji urządzeń i utraty funkcjonalności. W przypadku wszelkich problemów i pytań dotyczących tych grup po ich wywiązywce administratorzy IT mogą kontaktować się Microsoft Managed Desktop działaniami. Aby uzyskać więcej informacji, zobacz [Pomoc techniczna dla administratorów Microsoft Managed Desktop](../working-with-managed-desktop/admin-support.md). |

## <a name="your-roles-and-responsibilities"></a>Role i obowiązki użytkownika

Ten zestaw typowych ról i obowiązków jest wymagany do wdrożenia, ale nie jest zapewniany przez firmę Microsoft. Nie jest to wyczerpujące, ale ma zastosowanie w przypadku większości organizacji. Istnieje kilka elementów, za które odpowiada użytkownik i firma Microsoft.

| Rola lub odpowiedzialność | Opis |
| ----- | ----- |
| Zarządzanie zmianami | Firma Microsoft z wyprzedzeniem powiadomi klientów o zmianach w ich środowisku Microsoft Managed Desktop klienta. Aby uzyskać więcej informacji, zobacz [Zmiany w usłudze i komunikacja](../service-description/servicechanges.md).<br><br>Musisz mieć własny proces zarządzania zmianami i nawiązyć kontakt z zespołem Microsoft Managed Desktop Operations. Ponadto muszą być dostępne zasoby do przeglądania i zatwierdzania tych zmian. Aby uzyskać więcej informacji, zobacz [Operacje i monitorowanie](../service-description/operations-and-monitoring.md). |
| Zarządzanie tożsamościami | Odpowiadasz za tworzenie kont użytkowników, przypisywanie użytkowników do grup i utrzymywanie aktualnych metadanych. |
| Aplikacje Microsoft 365 dla przedsiębiorstw i zarządzanie | Firma Microsoft jest odpowiedzialna za zapewnienie Office aplikacje są wdrażane u użytkowników, a te aplikacje są zawsze aktualne. <br><br> Odpowiadasz za zarządzanie Microsoft 365 i zasadami, w tym Exchange Online administracji:<br><ul><li>Administrowanie pocztą e-mail</li><li> Konfiguracja skrzynki pocztowej i reguły</li><li>Exchange zarządzania lokalnego</li></ul><br>Odpowiadasz również za narzędzia do współpracy, zarządzanie SharePoint serwerami, zarządzanie domenami oraz zasady zabezpieczeń i informacji ustawione w centrum centrum administracyjne platformy Microsoft 365. |
| Pomoc techniczna dla użytkowników | Zapewnij wszystkim pomoc techniczną i pomoc techniczną od pierwszego kontaktu przez rozwiązanie problemu dla użytkownika przez Ciebie lub za pośrednictwem wyznaczonego partnera pomocy technicznej. Aby zapewnić pomoc techniczną dla tych obszarów, musisz bezpośrednio udzielić pomocy technicznej dla użytkowników lub współpracować z partnerem: <br><ul><li>Infrastruktura lokalna: cała łączność sieciowa i internetowa, infrastruktura sieci VPN i konfiguracja klienta, lokalny sprzęt do sal konferencyjnych, drukarki, serwer proxy i konfiguracja oraz zapory.</li><li>Zasoby chmury dla całej firmy: poczta SharePoint e-mail, usługi współpracy i inna infrastruktura chmury związana z technologiami w całej firmie.</li><li>Firma i wszelkie inne aplikacje specyficzne dla firmy.</li></ul>
| Aplikacje | Role i obowiązki różnią się nieco w zależności od poszczególnych aplikacji Microsoft Managed Desktop w porównaniu z aplikacjami, które ty zapewniasz. <br><br>W przypadku aplikacji dostarczanych przez firmę Microsoft (Aplikacje Microsoft 365 dla przedsiębiorstw na przykład programy Word, Excel, PowerPoint, Outlook, Publisher, Access, Skype dla firm, Teams i OneNote), **firma Microsoft** udostępni pełną obsługę wdrażania, aktualizacji i pomocy technicznej. **Musisz** uzyskać i przypisać licencje dla tych aplikacji, dodać użytkowników do grup zabezpieczeń, zarządzać końcem użytkowania i wdrożyć wszystkie potrzebne dodatki.<br><br>W przypadku aplikacji ( takich jak aplikacje firmowe) za te działania odpowiadasz zarówno w przypadku samodzielnie, jak i w przypadku zaangażowania dostawcy spoza firmy Microsoft:  <br><ul><li>Identyfikowanie aplikacji potrzebnych dla grup użytkowników docelowych</li><li>Tworzenie grup usługi Azure AD do wdrażania aplikacji i zarządzanie nimi</li><li>Pakowanie aplikacji w celu spełnienia Microsoft Intune standardów wdrażania</li><li> Przekazywanie aplikacji do usługi Microsoft Intune</li><li>Testowanie aplikacji w Microsoft Managed Desktop aplikacji</li><li>Testowanie aplikacji z użytkownikami</li><li>Zarządzanie użytkownikami i przypisywanie ich do aplikacji</li><li>Identyfikowanie i wdrażanie aktualizacji aplikacji za pośrednictwem Microsoft Intune</li><li>Odinstalowywanie i usuwanie aplikacji po ich wycofaniu</li><li>Powtarzanie i przypisywanie licencji</li><li>Świadczenie pomocy technicznej dla aplikacji biznesowych</li><li> Zdalne zarządzanie ustawieniami aplikacji</li></ul><br>**Firma Microsoft** udostępni Microsoft Intune wdrażania w celu dostarczania aplikacji klientom zdalnym.<br><br>Aby uzyskać więcej informacji, zobacz [Aplikacje](../get-ready/apps.md).
| Monitorowanie zabezpieczeń i reagowanie na nie | Odpowiadasz za badanie i rozwiązywanie zdarzeń związanych z urządzeniami, które nie Microsoft Managed Desktop urządzenia. Musisz upewnić się, że zespół Microsoft Managed Desktop operacyjny jest poinformowany o wszelkich problemach, które mogą mieć wpływ na usługę.
| Obsługa operacji | Musisz podać listę preferowanych kontaktów i ekspertów w danej dziedzinie w organizacji. Te kontakty są potrzebne w razie zdarzenia operacyjnego niezwiązanego z Microsoft Managed Desktop. <br><br>Odpowiadasz również za badanie i rozwiązywanie zdarzeń związanych z urządzeniami i usługami, które nie są w Microsoft Managed Desktop. Musisz zapewnić, że zespół operacyjny Microsoft Managed Desktop jest zawsze poinformowany.
| Infrastruktura sieci, w tym połączenia VPN | Odpowiadasz za konfigurację, konfigurację i zarządzanie (w tym rozwiązywanie problemów i debugowanie) wszystkich usług i infrastruktury związanej z siecią. Obejmuje to również łączność z Internetem, kontrolki sieciowe, konfigurację serwera proxy i infrastrukturę łączności zdalnej.<br><br>Jeśli serwer proxy jest skonfigurowany (w sprzęcie lub oprogramowaniu), istnieje kolekcja adresów URL, które muszą być dozwolone przez ten serwer proxy. Odpowiadasz za rozwiązywanie wszelkich konfliktów i niezgodności spowodowanych wieloma komputerami proxy. Możesz dodać sieciowe proxy specyficzne dla Twojej organizacji przy użyciu konfigurowalnych ustawień. Aby uzyskać więcej informacji, zobacz [Konfigurowanie ustawień](../working-with-managed-desktop/config-setting-ref.md#proxy).<br><br>Aby uzyskać więcej informacji, zobacz [Konfiguracja serwera proxy](../get-ready/network.md).
| Drukowanie | Ponosisz odpowiedzialność za instalowanie, konserwację i administrowanie drukarkami i kolejkami wydruku. Drukowanie w chmurze jest zalecanym rozwiązaniem, ale nie jest wymagane.
