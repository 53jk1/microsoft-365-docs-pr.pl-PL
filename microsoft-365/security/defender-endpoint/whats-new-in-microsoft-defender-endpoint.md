## <a name="september-2021"></a>Wrzesień 2021

- [Filtrowanie zawartości sieci Web](web-content-filtering.md) <br/>W ramach funkcji ochrony sieci Web w programie Microsoft Defender for Endpoint filtrowanie zawartości sieci Web umożliwia zespołowi zabezpieczeń Twojej organizacji śledzenie i regulowanie dostępu do witryn internetowych na podstawie ich kategorii zawartości. Kategorie obejmują zawartość dla dorosłych, wysoką przepustowość, odpowiedzialność prawnie, rozrywka i niekategoryzowane. Wiele witryn internetowych, które należą do jednej lub większej liczby kategorii, może nie być złośliwych, jednak może być problematycznych z powodu przepisów dotyczących zgodności z przepisami, wykorzystania przepustowości lub innych kwestii. [Dowiedz się więcej o filtrowaniu zawartości sieci Web](web-content-filtering.md).

## <a name="june-2021"></a>Czerwiec 2021

- [Ocena luk w zabezpieczeniach oprogramowania eksportu różnicowego](get-assessment-methods-properties.md#31-methods) API <br> Dodatek do oceny eksportu [luk i bezpiecznego zbioru](get-assessment-methods-properties.md) interfejsu API konfiguracji. <br> W przeciwieństwie do pełnej oceny luk w oprogramowaniu (odpowiedź JSON) — która służy do uzyskania pełnej migawki oceny luk w oprogramowaniu organizacji za pomocą urządzenia — połączenie interfejsu API eksportu różnicowego jest używane do uzyskiwania zdalnego dostępu tylko do zmian, które wydarzyły się między wybraną datą a bieżącą datą (wywołaniem interfejsu API "delta"). Zamiast pełnego eksportu z dużą ilością danych za każdym razem będziesz otrzymywać tylko określone informacje dotyczące nowych, stałych i zaktualizowanych luk w zabezpieczeniach. Za pomocą wywołania interfejsu API eksportu różnicowego można również obliczać różne wskaźniki KPI, na przykład "ile luk usunięto" lub "ile nowych luk dodano do organizacji".

- [Eksportowanie ocen luk i bezpiecznych konfiguracji](get-assessment-methods-properties.md) API <br> Dodaje kolekcję interfejsów API, które ściągają Zarządzanie zagrożeniami i lukami danych na urządzenia. Istnieją różne wywołania interfejsu API, aby uzyskać różne typy danych: bezpieczna ocena konfiguracji, ocena zapasów oprogramowania i ocena luk w oprogramowaniu. Każde wywołanie interfejsu API zawiera dane wymagane dla urządzeń w organizacji.

- [Działania naprawcze](get-remediation-methods-properties.md) API <br> Dodaje zbiór interfejsów API z odpowiedziami, które zawierają Zarządzanie zagrożeniami i lukami działania naprawcze utworzone w dzierżawie. Typy informacji dotyczących odpowiedzi obejmują jedno działanie naprawcze według identyfikatora, wszystkie działania naprawcze i dostępne urządzenia w ramach jednego działania naprawczego.

- [Odnajdowanie urządzeń](device-discovery.md) <br> Ułatwia znajdowanie nieza zarządzania urządzeniami połączonymi z siecią firmową bez konieczności dodatkowych urządzeń czy uciążliwych zmian w procesach. Korzystając z urządzeń podłączonych do urządzenia, możesz znaleźć urządzenia niezakierowane w twojej sieci oraz oceniać luki w zabezpieczeniach i zagrożenia. Następnie możesz w nim wychować wykryte urządzenia, aby zmniejszyć ryzyko związane z posiadaniem nieza zarządzania punktami końcowymi w Twojej sieci.

   > [!IMPORTANT]
   > Odnajdowanie standardowe będzie trybem domyślnym dla wszystkich klientów począwszy od 19 lipca 2021 r. Możesz zachować tryb podstawowy za pośrednictwem strony ustawień.

- [Definicje grup urządzeń](/microsoft-365/security/defender-endpoint/machine-groups) mogą teraz zawierać wiele wartości dla każdego warunku. Możesz ustawić wiele tagów, nazw urządzeń i domen na definicję jednej grupy urządzeń.

- [Obsługa zarządzania aplikacją mobilną](https://techcommunity.microsoft.com/t5/microsoft-defender-for-endpoint/announcing-new-capabilities-on-android-and-ios/ba-p/2442730) <br> To rozszerzenie umożliwia programowi Microsoft Defender for Endpoint ochronę danych organizacji w obrębie zarządzanej aplikacji, gdy usługa Intune jest używana do zarządzania aplikacjami mobilnymi. Aby uzyskać więcej informacji na temat zarządzania aplikacją mobilną, zobacz [tę dokumentację](/microsoft-365/mem/intune/apps/mam-faq).

- [Microsoft Tunnel vpn](https://techcommunity.microsoft.com/t5/microsoft-defender-for-endpoint/announcing-new-capabilities-on-android-and-ios/ba-p/2442730) <br> Microsoft Tunnel sieci VPN są teraz zintegrowane z aplikacją Microsoft Defender for Endpoint dla systemu Android. Ta unfikacja umożliwia organizacjom oferowanie użytkownikowi końcowemu uproszczonego korzystania z jednej aplikacji zabezpieczającej — oferując zarówno obronę przed zagrożeniami na urządzeniach przenośnych, jak i możliwość dostępu do zasobów w premedytację z ich urządzeń przenośnych, a także zespoły zabezpieczeń i IT mogą utrzymywać te same funkcje administracyjne, które znajomy.

- [Wykrywanie błędów w systemie iOS](/microsoft-365/security/defender-endpoint/ios-configure-features#conditional-access-with-defender-for-endpoint-on-ios) <br> Funkcja wykrywania błędów w programie Microsoft Defender for Endpoint w systemie iOS jest teraz ogólnie dostępna. Powoduje to dodanie ochrony przed wyłudzaniem informacji, która już istnieje.  Aby uzyskać więcej informacji, zobacz [Konfigurowanie zasad dostępu warunkowego na podstawie sygnałów ryzyka urządzenia](/microsoft-365/security/defender-endpoint/ios-configure-features).
